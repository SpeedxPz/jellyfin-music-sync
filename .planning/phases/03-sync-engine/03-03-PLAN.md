Create src/main/lib/sync-engine.ts. Follow this architecture exactly:

```
SyncEngineOpts {
  playlistIds: string[]
  destination: string
  concurrentDownloads: number  // 1-5
}

runSync(opts, webContents, signal):
  1. scan destination for *.part files → delete each (D-ERR-ORPHANS)
  2. safeReadManifest = readManifest(destination)
  3. for each playlistId in opts.playlistIds:
       fetch all tracks via getPlaylistsApi paginated (limit=500, startIndex loop)
  4. build unified allTracksMeta: Map<itemId, BaseItemDto>
  5. build toDownload: items where needsDownload(manifest, id, destination) === true
  6. build toDelete: for each playlistId, items in manifest.playlists[playlistId]?.itemIds that
       are no longer in Jellyfin's current list AND NOT isReferencedByOtherPlaylist(manifest, id, opts.playlistIds)
       (D-DEL-SCOPE, D-DEL-ABANDONED: only delete from currently-selected playlists)
  7. delete toDelete items: unlinkSync(join(destination, ...localPath.split('/'))); delete from manifest.items
  8. p-limit(opts.concurrentDownloads) download queue for toDownload:
       - buildLocalPath + buildDownloadUrl
       - downloadTrack with onChunk → webContents.send('sync:onProgress', SyncProgress{status:'downloading'})
       - on success: update manifest.items[id] with localPath (forward-slash), fileSize, syncedAt
                     emit SyncProgress{status:'complete'}; summary.added++
       - on AbortError: re-throw (cancellation propagates)
       - on other error: summary.failed++; summary.failures.push({name, reason}); emit SyncProgress{status:'error'}
  9. for each playlistId, update manifest.playlists[playlistId] with current itemIds
  10. for each playlistId, write M3U8 file:
        content = generateM3u8(manifestPlaylist, m3u8ItemsMap, destination)
        writeFileSync(join(destination, sanitizePathSegment(playlistName)+'.m3u8'), content, 'utf-8')
  11. writeManifest(destination, manifest)
  12. return SyncSummary
```

Key implementation requirements:

**Orphan scan (step 1):** Use `globSync` or manual `readdirSync` + filter for `*.part` pattern. Use `readdirSync` recursively — `*.part` files can be in subdirectories. Pattern:
```typescript
import { readdirSync, statSync, unlinkSync } from 'fs'
function deleteOrphanedPartFiles(root: string): void {
  try {
    const entries = readdirSync(root, { withFileTypes: true })
    for (const entry of entries) {
      const fullPath = join(root, entry.name)
      if (entry.isDirectory()) {
        deleteOrphanedPartFiles(fullPath)  // recurse
      } else if (entry.name.endsWith('.part')) {
        try { unlinkSync(fullPath) } catch { /* ignore */ }
      }
    }
  } catch { /* ignore if root doesn't exist yet */ }
}
```

**Pagination (step 3):** Mirror the pattern from playlists.ts exactly:
```typescript
import { getPlaylistsApi } from '@jellyfin/sdk/lib/utils/api/playlists-api'
import { ItemFields } from '@jellyfin/sdk/lib/generated-client/models'
// ...
while (true) {
  const resp = await getPlaylistsApi(api).getPlaylistItems({
    playlistId,

    startIndex,
    limit: 500,
    fields: [ItemFields.MediaSources],
  })
  const items = resp.data.Items ?? []
  for (const item of items) { if (item.Id) playlistTracks.push(item) }
  if (items.length === 0) break
  startIndex += 500
}
```

**localPath storage (manifest):** Always store as forward slashes:
```typescript
const absPath = buildLocalPath(destination, artist, album, name, container)
const relPath = relative(destination, absPath).split(sep).join('/')  // forward slashes
// store relPath in manifest.items[id].localPath
```

**p-limit import (CJS v3):**
```typescript
import pLimit from 'p-limit'
const limit = pLimit(opts.concurrentDownloads)
const tasks = toDownload.map((item) => limit(() => downloadOne(item)))
await Promise.allSettled(tasks)
```

**Cancellation:** Pass `signal` to `downloadTrack`. When axios throws an AbortError (`err.name === 'CanceledError'` or `err.name === 'AbortError'` or `signal.aborted`), stop queueing new downloads and return the partial summary. Do NOT count aborted tracks as failures — they are simply incomplete.

**M3U8 items map (step 10):** Build `Record<string, M3u8ItemMeta>` from manifest.items filtered to this playlist's itemIds. The `runTimeTicks` and `name` must come from the Jellyfin metadata fetched in step 3 (not stored in manifest). Keep a `Map<itemId, { runTimeTicks, name }>` in memory during the run.

**D-DEL-ABANDONED:** Only attempt deletion for items that belong to playlists in `opts.playlistIds`. Items in manifest playlists NOT in `opts.playlistIds` are left completely untouched.

**File extension:** Use `item.Container ?? 'mp3'` as fallback. Container is typically 'flac', 'mp3', 'ogg', 'aac' — already available on BaseItemDto without extra fields, but `ItemFields.MediaSources` is requested anyway for mediaSourceId.

**webContents guard:** Before every `webContents.send()` call, check `!webContents.isDestroyed()`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Jellyfin API response → sync-engine.ts | Track metadata (names, artists, albums) is untrusted external input |
| Manifest on disk → sync-engine.ts | Manifest may be corrupt or from a different version |
| AbortSignal → axios requests | Cancellation signal crosses the download pipeline boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Tampering | deleteOrphanedPartFiles in sync-engine.ts | mitigate | Only deletes files ending in `.part` within destination root; recursive scan is bounded to destRoot; does not traverse symlinks aggressively |
| T-03-03-02 | Tampering | toDelete path construction in sync-engine.ts | mitigate | localPath stored with forward slashes; reconstructed via `join(destination, ...entry.localPath.split('/'))` — prevents path escape via backslash injection; `isReferencedByOtherPlaylist` checked before every deletion |
| T-03-03-03 | Denial of Service | Pagination loop in fetchPlaylistTracks | accept | Loop terminates on empty page response; Jellyfin server controls total item count; bounded by server response; no infinite loop risk for well-behaved server |
| T-03-03-04 | Information Disclosure | SyncProgress event via webContents.send | accept | Progress events contain track names (not auth tokens or file content); renderer is same-origin Electron window; contextIsolation prevents script injection |
| T-03-03-05 | Tampering | M3U8 write via writeFileSync | mitigate | M3U8 filename derived from `sanitizePathSegment(playlistName)` — prevents directory traversal via playlist name containing `../`; written to destRoot only |
</threat_model>
