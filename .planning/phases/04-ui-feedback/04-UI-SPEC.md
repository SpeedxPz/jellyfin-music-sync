## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, inline padding (`gap-1`) |
| sm | 8px | Compact element spacing, label-to-input gap (`gap-2`, `space-y-2`) |
| md | 16px | Default element spacing, section gaps (`gap-4`, `space-y-4`) |
| lg | 24px | Main content padding, section padding (`p-6`, `px-6`) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Vertical centering offset, hero spacing |
| 3xl | 64px | Page-level spacing (not used in phase 4 narrow layouts) |

Exceptions:
- Touch targets: minimum 44px height on all interactive elements (established pattern from `LoginScreen.tsx` — `min-h-[44px]`). This is not a spacing token but a minimum interactive size constraint.
- Header bar vertical padding: `py-3` (12px) — matches existing `PlaylistBrowserScreen.tsx` header. Acceptable deviation for compact header chrome.
- Progress bar height: 8px (`h-2`) for overall bar; 6px (`h-1.5`) for per-file bar. Sub-grid values intentional for visual differentiation.

Source: Measured from `PlaylistBrowserScreen.tsx` (`px-6 py-3`, `p-6 gap-4`) and `LoginScreen.tsx` (`p-6`, `space-y-4`).

## Color

| Role | Value | Tailwind | Usage |
|------|-------|----------|-------|
| Dominant (60%) | #111827 | `bg-gray-900` | Full-screen background, page surface |
| Secondary (30%) | #1f2937 | `bg-gray-800` | Header bars, card surfaces, input fields (`bg-gray-700` for inputs) |
| Accent (10%) | #2563eb | `bg-blue-600` | Reserved list below |
| Destructive | #f87171 | `text-red-400` | Error text, failed-track count only |

Accent (`blue-600` / `#2563eb`) reserved for:
1. Primary CTA button ("Sync Selected" in `PlaylistBrowserScreen.tsx`)
2. Input focus ring (`focus:border-blue-500`)
3. Checkbox accent (`accent-blue-500`)
4. Loading spinner ring (`border-t-blue-400`)

Accent is NOT used for: informational text, headers, progress bar fill (see below), counter rows, or navigation.

Progress bar fill color: `bg-blue-500` — same hue family as accent, 1 shade lighter to distinguish from buttons. Progress bars are state indicators, not interactive elements.

Additional semantic colors (informational, not destructive):
- `text-gray-400` (#9ca3af) — secondary/muted text, counters, timestamps
- `text-gray-300` (#d1d5db) — subheadings, slightly de-emphasized headings
- `text-yellow-300` / `bg-yellow-900/40` — Linux plaintext warning banner (established, carry forward)
- `text-green-400` — success indicators (✔ added count in summary screen only)
- `text-red-400` — failed/error indicators (✖ failed count, error messages)
- `text-gray-400` — unchanged/skipped count (⏭ unchanged, ⧖ remaining)

Source: Derived from `LoginScreen.tsx` and `PlaylistBrowserScreen.tsx` color classes. Green and muted gray for summary counts are phase 4 additions that extend the established palette.

## Screen States

### SyncScreen states

| State | Trigger | Rendering |
|-------|---------|-----------|
| Active — no progress yet | `syncState = 'syncing'`, no events received | Show bars at 0%, "Now: —", counters at 0 |
| Active — downloading | `sync:progress` events arriving | Live update all values |
| Stopping | User clicked Stop Sync (before `sync:complete` fires) | Disable Stop Sync button, label changes to "Stopping…", bars frozen at last value |
| Transitioning to summary | `sync:complete` fires | Unmount SyncScreen, mount SyncSummaryScreen |

### SyncSummaryScreen states

| State | Trigger | Rendering |
|-------|---------|-----------|
| Complete | `canceled = false` | "Sync Complete" heading, green ✔ count |
| Canceled | `canceled = true` | "Sync Canceled" heading, no green on ✔ (gray-400), ⧖ remaining shown |
| Failures collapsed | `failures.length > 0`, initial | "✖ N failed — show details ▾" visible, list hidden |
| Failures expanded | User clicked toggle | Full failures list visible, toggle reads "▴ hide details" |
| No failures | `failures.length === 0` | Failures section not rendered at all |

## Interaction Contracts

### Stop Sync button behavior
- Single click fires `window.electronAPI.sync.cancel()` (no confirmation modal).
- Button immediately enters disabled + "Stopping…" text state.
- Button re-enables only if the user somehow reaches SyncScreen again (new sync).
- Source: D-CANCEL.

### Failures expand/collapse
- Local `useState<boolean>(false)` controls visibility.
- No animation (D-FAILURES: "simple show/hide").
- Toggle button switches text between collapsed and expanded copy.
- Section absent entirely when `failures.length === 0`.

### Downloads control
- Reads initial value from `window.electronAPI.settings.get()` on PlaylistBrowserScreen mount.
- − button disabled when value === 1; + button disabled when value === 5.
- Each click calls `window.electronAPI.settings.set({ concurrentDownloads: N })`.
- Value passed into `SyncOptions.concurrentDownloads` when starting sync (replaces hardcoded `3`).
- Source: D-SETTINGS-CONTROL, D-SETTINGS-PASS.

### Progress event subscription
- Subscribe `window.electronAPI.on('sync:progress', cb)` on SyncScreen mount.
- Subscribe `window.electronAPI.on('sync:complete', cb)` on SyncScreen mount.
- Both subscriptions in a single `useEffect` with no cleanup dependency (fire-and-forget; Electron IPC listeners are app-lifespan).
- Source: D-PROG-EVENTS.

### Open destination folder
- Calls a new IPC handler `shell:openPath` (or inline on `sync` namespace) passing stored destination path.
- Button disabled if destination is empty string (edge case guard).
- Source: POST-03, D-SUMMARY-DESTINATION.
