## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Tailwind class | Usage |
|-------|-------|---------------|-------|
| xs | 4px | p-1 / gap-1 | Icon gaps, inline badge padding |
| sm | 8px | p-2 / gap-2 | Compact element spacing, input-icon gap |
| md | 16px | p-4 / gap-4 | Default element spacing, form row gaps |
| lg | 24px | p-6 / gap-6 | Card/panel padding, section padding |
| xl | 32px | p-8 / gap-8 | Layout gaps, screen-level padding |
| 2xl | 48px | py-12 | Major vertical breaks |
| 3xl | 64px | — | Not used this phase |

Exceptions:
- Password show/hide toggle button: minimum 44px touch target height (`min-h-[44px]`) — accessibility requirement for interactive controls.
- Checkbox targets in playlist list: minimum 44px row height (`min-h-[44px]`) for pointer accuracy on desktop.

Source: 8-point grid default; exceptions are standard WCAG 2.5.5 minimum target size.

## Typography

| Role | Size | Weight | Line Height | Tailwind classes |
|------|------|--------|-------------|-----------------|

- Maximum 4 sizes — no intermediate sizes permitted this phase.

Source: Phase 1 App.tsx established `text-sm`, `font-semibold`, `font-medium` pattern; sizes extrapolated to cover Login + Playlist Browser screens.

## Color

All colors are Tailwind CSS v4 utility classes drawn from the gray palette established in Phase 1 App.tsx (`bg-gray-900`, `text-gray-100`).

| Role | Tailwind value | Approx hex | Usage |
|------|---------------|-----------|-------|
| Dominant (60%) | `bg-gray-900` | #111827 | App window background, Login screen background |
| Secondary (30%) | `bg-gray-800` | #1f2937 | Login card surface, Playlist Browser panel surface |
| Tertiary surface | `bg-gray-700` | #374151 | Filter input background, list row hover state |
| Text primary | `text-gray-100` | #f3f4f6 | All primary readable text |
| Text secondary | `text-gray-400` | #9ca3af | Placeholders, track counts, version string, helper text |
| Border | `border-gray-600` | #4b5563 | Card borders, input borders, dividers |
| Accent (10%) | `bg-blue-600` / `text-blue-400` | #2563eb / #60a5fa | Connect button (bg-blue-600 hover:bg-blue-700), selected playlist checkbox checked state, server reachability success indicator |
| Destructive | `text-red-400` / `bg-red-900/30` | #f87171 / — | Inline error messages below Connect button; Logout button text (text-red-400, no background) |

Accent reserved for:
1. Connect button background (`bg-blue-600`, `hover:bg-blue-700`)
2. Checkbox checked state in playlist list
3. Server reachability success indicator ("Reachable: Jellyfin X.Y.Z" prefix icon or text color)

No other elements may use blue. Destructive red is reserved for error messages and Logout button label only.

Source: Phase 1 App.tsx codebase scan (gray-900/800/600 palette); D-AUTH-LINUX (Linux warning banner); D-ERROR-DISPLAY (inline error); D-SERVER-VALIDATE (reachability indicator).

## Screen Layouts

### Login Screen

```
┌─────────────────────────────────────┐  bg-gray-900 (full screen)
│                                     │
│   ┌─────────────────────────────┐   │  bg-gray-800, rounded-lg, border-gray-600
│   │  Jellyfin Music Sync   (28px semibold, centered)              │
│   │  ─────────────────────────  │
│   │  Server URL                 │  text-sm label
│   │  [________________________] │  input, border-gray-600, bg-gray-700
│   │                             │
│   │  Username                   │  text-sm label
│   │  [________________________] │
│   │                             │
│   │  Password                   │  text-sm label
│   │  [____________________][👁] │  show/hide toggle (44px target)
│   │                             │
│   │  [──── Connect ──────────]  │  bg-blue-600, full-width, text-white
│   │                             │
│   │  ✓ Reachable: Jellyfin 10.9 │  text-blue-400 text-sm (shown after ping)
│   │  ✗ Error message here       │  text-red-400 text-sm (shown on error)
│   └─────────────────────────────┘
│                                     │
└─────────────────────────────────────┘
```

- Card: max-w-sm, centered via flex items-center justify-center, p-6 internal padding.
- Connect button: full-width within card, py-2, font-semibold, rounded.
- Reachability indicator and error both render inline below Connect button in the same reserved vertical slot. Only one shows at a time.
- Three error variants (source: D-SERVER-VALIDATE):
  1. "Could not reach server. Check the URL and try again."
  2. "URL reached but not a Jellyfin server. Is the URL correct?"
  3. "Login failed. Check your username and password."
- Input fields remain populated on error (source: D-ERROR-DISPLAY).

### Playlist Browser Screen

```
┌─────────────────────────────────────┐  bg-gray-900 (full screen)
│  Jellyfin Music Sync   [Log out]    │  Header bar, bg-gray-800, border-b-gray-600
│─────────────────────────────────────│
│  Logged in as: {username}           │  text-sm text-gray-400
│  {server name}                      │
│─────────────────────────────────────│
│  Your Playlists             (20px)  │  Heading
│  [Filter by name...        ]        │  bg-gray-700 input, full-width
│─────────────────────────────────────│
│  ☐  Playlist Name A    42 tracks    │  min-h-[44px] row, hover:bg-gray-700
│  ☑  Playlist Name B   108 tracks    │  checked = blue checkbox
│  ☐  Playlist Name C     7 tracks    │
│  ...                                │
│─────────────────────────────────────│
│  3 playlists selected               │  text-sm text-gray-400
│  [────────── Sync Selected ────────] │  bg-blue-600, full-width or fixed-bottom
└─────────────────────────────────────┘
```

- Header bar: full-width, bg-gray-800, border-b, px-6 py-3. Contains app name (left) and Logout button (right, text-red-400, no background, hover:text-red-300).
- Filter input: full-width within content area, bg-gray-700, border-gray-600, placeholder "Filter by name...".
- Playlist rows: full-width, px-4, min-h-[44px], flex items-center gap-3. Checkbox + playlist name (text-sm font-semibold) + track count (text-sm text-gray-400, right-aligned).
- Selection count footer: sticky at bottom of list area, text-sm text-gray-400, left-aligned.
- Sync Selected button: full-width, bg-blue-600, py-2, font-semibold. Disabled + opacity-40 when zero playlists selected.

### Linux Warning Banner (conditional)

- Full-width bar, bg-yellow-900/40, border-y border-yellow-700, text-yellow-300, text-sm, px-6 py-2.
- Persistent — no close/dismiss button.

## Interaction Contracts

### Login Screen

| Interaction | Behavior |
|-------------|----------|
| User types in any field | No validation on keystroke — validate only on Connect click |
| Connect clicked | 1. Disable button + show "Connecting..." 2. Ping `/System/Info/Public` 3a. On timeout/ECONNREFUSED → show error 1, re-enable button 3b. On non-Jellyfin response → show error 2, re-enable button 3c. On Jellyfin confirmed → show "Reachable: {name}" → proceed to login 4. POST credentials to Jellyfin auth 5a. On 401 → show error 3, re-enable button 5b. On success → store token → transition to Playlist Browser |
| Error shown | Input fields remain populated. Error text appears below Connect button. |
| Reachability confirmed | "Reachable: {serverName} {version}" shown in blue below Connect button, immediately before login attempt proceeds. |
| Password show/hide | Toggle eye icon button. Switches `input type` between `password` and `text`. Aria-label updates accordingly. |

### Playlist Browser Screen

| Interaction | Behavior |
|-------------|----------|
| Screen mounts | Fetch all playlists (paginated, limit=500 loop). Show spinner during fetch. |
| Filter input changes | Immediate client-side filter on every keystroke. Case-insensitive substring match on playlist name. No debounce needed (local data). |
| Filter clears all results | Show empty state: "No playlists match your filter. Try a different name." |
| Playlist row clicked (anywhere in row) | Toggle checkbox for that playlist. |
| Checkbox toggled | Update selection state. Update selection count footer. |
| Zero playlists selected | "Sync Selected" button disabled (`opacity-40 cursor-not-allowed`). |
| One or more selected | "Sync Selected" button enabled. |
| Sync Selected clicked | Out of scope for Phase 2 — button exists but triggers no action yet (Phase 3 wires it). Button may show a placeholder alert or do nothing. |
| Log out clicked | Immediately: clear stored token (call `auth:logout` IPC), return to Login screen. No confirmation dialog. |

## States Reference

### Login Screen States

| State | Description |
|-------|-------------|
| Idle | All fields empty, Connect enabled, no error shown |
| Pinging | Connect disabled ("Connecting..."), no error shown |
| Reachable | "Reachable: {server}" shown in blue, auto-transitions to Signing in |
| Signing in | Connect disabled ("Signing in...") |
| Error | Error text shown below Connect, button re-enabled, fields populated |
| Authenticated | Screen unmounts, Playlist Browser mounts |

### Playlist Browser States

| State | Description |
|-------|-------------|
| Loading | Spinner + "Loading playlists..." shown in place of list |
| Loaded, no selection | Full list shown, Sync button disabled |
| Loaded, filtered | Filtered list shown, selection preserved for unfiltered items |
| Loaded, filter empty result | Empty state copy shown |
| Loaded, with selection | Selection count shown, Sync button enabled |
| Logged out | Screen unmounts, Login screen mounts |
